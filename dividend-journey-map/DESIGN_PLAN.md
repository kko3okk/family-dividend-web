# DESIGN_PLAN — 股息里程地圖

## Color

配色以「夜間長途跋涉」為母題：這不是一張報表，是一張深夜還在走的路。
背景用低彩度的靛藍夜色，把亮度預算全部留給路徑與站點——因為使用者只需要
在 3 秒內找到兩件事：**我在哪裡、下一站在哪裡**。

| 具名色 | Hex | 語意 |
|--------|-----|------|
| `--sky-far` | `#2C3358` | 最遠山層。彩度最低、明度最暗，退到最後面 |
| `--sky-mid` | `#39406A` | 中景山層 |
| `--sky-near` | `#474E82` | 近景山層。三層明度階差固定 11%，形成空氣透視 |
| `--ground` | `#1E2340` | 地面。比最遠山更暗，讓路徑浮起來 |
| `--path-base` | `#4A5280` | 未走過的路。與地面對比僅 1.8:1，刻意「不搶戲」 |
| `--path-done` | `#F2B544` | 已走過的路。全畫面彩度最高的暖黃，是唯一的光源色 |
| `--station-cleared` | `#5FBF8C` | 已抵達。綠 = 完成，配勾號 |
| `--station-active` | `#F2B544` | 進行中。與 path-done 同色，形成「光沿著路走到這裡」的連續感 |
| `--station-locked` | `#6E7599` | 未解鎖。灰藍，飽和度僅 18% |

**為什麼 active 與 path-done 同色**：讓「已走的路」與「現在的位置」共用一個
光源色，視線會沿著暖黃自然滑到當前站點——這是 3 秒理解的路徑設計，不是裝飾。

## Type

三種角色，字級尺標以 1.5 倍率遞進：

| 角色 | 字體 | 字級 | 用途 |
|------|------|------|------|
| display | Fredoka 600 | 46px（`--fs-display`） | 目前年股息數字。全畫面唯一的巨大數字 |
| station | Noto Sans TC 700 | 20px（`--fs-station`） | 站點名稱 |
| meta | Noto Sans TC 400 / Fredoka 400 | 13px（`--fs-meta`） | 完成度、預估年數、目標金額 |

數字一律走 Fredoka 並開啟 `font-variant-numeric: tabular-nums`——金額每天在變，
等寬數字才不會讓標籤卡寬度跳動。中文以 Noto Sans TC 為備援（jf open 粉圓 未在
Google Fonts 上架，需自行託管；本專案先以 Noto Sans TC 實作，字型檔到位後
只需改 `--font-body` 一個變數）。

## Layout

畫布 1200×700，橫向敘事。路徑從左下出發、向右上爬升——上升方向天然對應
「資產增長」，不需要額外說明。站點沿路徑分布，愈往右上目標愈大。
標籤卡固定掛在站點下方 24px，卡片間距不足時改掛上方。
左上角是唯一的巨大數字（目前年股息）與一句話進度說明。

## Signature

**「光沿著路爬升」**——已走過的路徑是一條發亮的暖黃線，從左下的起點一路延伸到
目前位置，末端停在會脈動的當前站點上；前方的路是暗的虛線。整張圖只有這一條線
是亮的，所以無論縮到多小、看多快，視線都會先落在「你走到哪裡」。

隨著年股息累積，這條光線會一年比一年長——這是使用者十四年間唯一需要感受的變化。

## 自我檢查修訂紀錄

**檢查問題**：「這份計畫換成任何一個理財產品都能套嗎？」

**初版答案：是。** 初版的 Color 段只寫了「深色背景配高彩度強調色，提升可讀性」，
Signature 寫的是「清楚的進度條」——這兩段套到記帳 App、保險試算、貸款計算機
都成立，等於什麼都沒說。

**修訂內容**：

1. **Color**：改為綁定本產品的特定判斷——「亮度預算全部留給路徑與站點」，
   並說明 `--station-active` 與 `--path-done` 刻意同色的理由（讓視線沿光線滑動）。
   這個決定只在「路徑型進度視覺化」成立，換成圓餅圖或長條圖就不適用。

2. **Signature**：從泛用的「進度條」改為「光沿著路爬升」，並綁定本產品的
   時間尺度——十四年、每年變長一點。這個 signature 換到任何短週期產品都不成立，
   因為短週期看不出「一年比一年長」的累積感。

3. **Layout**：明確寫出「左下往右上」的方向選擇理由（上升＝資產增長），
   而非泛用的「置中對齊、留白充足」。

**修訂後再自檢**：換成保險試算不成立（無多年累積路徑）、換成記帳 App 不成立
（無終點目標）、換成貸款計算機不成立（方向相反，是遞減）。通過。
